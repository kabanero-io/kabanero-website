Kabanero Developer Experience for CLI

The Kabanero Development Experience for CLI is simply the Appsody CLI. Appsody provides Kabanero with simplicity in the creation of cloud-native applications in containers. Appsody provides pre-configured container images (stacks) and project templates for a growing set of popular open source runtimes and frameworks, providing a foundation on which to build applications for Kubernetes and Knative deployments.

Check out this video that shows the developer experience for a Node.js app.

Appsody points to a growing community of Stacks (Container images prebuilt with runtimes and frameworks). Kabanero takes a subset of these stacks and augments them with build and deploy technology to create a Stack. Stacks enable Kabanero to build an application in a controlled manner. This controlled flow ensures that the application meets security requirements and deploys into a RHOCP (Red Hat OpenShift Container Platform) project. When using Appsody with Kabanero, you point Appsody at your Stack Hub.

Try it using the Appsody CLI

Kabanero uses Appsody as its CLI experience to access stacks. These instructions will get you starting using the CLI.

  1. Follow the instructions to install the Appsody CLI
  2. Add your Application Stacks Hub Repository. It helps to add the version to the repo name. For example, in the command below you can use kabanero<VERSION> as the name
    • appsody repo add kabanero <YOUR_STACK_HUB_URL>
      • If you do not have a stack hub URL, you can use the latest public Application Stack Hub URL:

        https://github.com/kabanero-io/collections/releases/latest/download/kabanero-index.yaml

  3. Optional - List your available stacks
    • appsody list kabanero
  4. Create an application
    • Create a new project directory and cd into it
    • appsody init kabanero/STACK_ID (STACK_ID can be seen from the appsody list command above)
